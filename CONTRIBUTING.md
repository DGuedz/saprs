# Contributing to SAPRS

SAPRS is an open specification. Contributions are welcome from any Solana ecosystem builder.

## How to Contribute

### Feedback on SAPRS-001

Open a GitHub issue with label `spec-feedback`. Include:
- The section you're commenting on
- The problem or improvement
- Your proposed change (if any)
- Your project context (optional but helpful)

### Propose an Extension

New SAPRS specs (SAPRS-002+) follow this process:

1. Open an issue titled `[PROPOSAL] SAPRS-00X: <topic>`
2. Include: problem statement, proposed schema changes, ecosystem impact
3. Discussion period: 2 weeks minimum
4. If accepted: PR with spec document in `specs/`

### Adopt SAPRS

If your project implements SAPRS-001, open a PR to add yourself to `ADOPTERS.md`.

## Standards

- Specs are written in Markdown
- JSON examples use 2-space indent
- YAML examples follow the existing schema conventions
- All field names use `snake_case`
- Breaking changes require a new spec version
